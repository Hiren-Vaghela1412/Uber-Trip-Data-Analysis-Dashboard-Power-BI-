# 🚖 Uber Trip Data Analysis Dashboard (Power BI)

## 📌 Project Overview
This project showcases an interactive Power BI dashboard designed to analyze Uber trip data. It helps uncover meaningful insights related to booking patterns, revenue performance, trip behavior, and demand trends.

The dashboard is built to support data-driven decision-making through clear visualizations and user-friendly interactivity.

---

## 🎯 Project Goals
- Analyze overall booking and revenue performance
- Understand trip distance and duration behavior
- Identify peak demand time periods
- Explore location-based trends (pickup & drop-off)
- Compare performance across vehicle categories

---

## 📊 Dashboard Sections

### 🔹 1. Summary Dashboard
Provides a high-level overview of key metrics:

**Key Metrics:**
- Total Trips
- Total Revenue
- Average Fare
- Total Distance Covered
- Average Trip Distance
- Average Trip Duration

**Visual Insights:**
- Trips by Payment Type
- Day vs Night Trip Distribution
- Daily Booking Trends
- Vehicle Type Comparison
- Top Pickup & Drop Locations

## ⚙️ Key Features

### 🔄 Dynamic Metrics
- Implemented dynamic measure selection using **SWITCH()**
- Users can toggle between key KPIs

### 🔗 Data Modeling
- Created relationships between tables
- Used **USERELATIONSHIP()** for advanced analysis

### 🎛️ Interactivity
- Slicers (Date, City, Vehicle Type)
- Dynamic titles based on selection
- Drill-through navigation
- Bookmark for better user experience

---

## 📈 Key Insights
- Peak demand occurs during daytime working hours
- Weekend bookings are higher than weekdays
- Majority of trips are short-distance
- Some locations consistently generate high traffic
- One vehicle category dominates user preference
